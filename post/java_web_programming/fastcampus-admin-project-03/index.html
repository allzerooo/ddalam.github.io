URL Mapping

GetController 클래스와 PostCotroller 클래스에 똑같이 @RequestMapping("/api")를 쓰는 것은 상관없다. 주소가 겹치는 것은 클래스 내 메서드에 단위로 판단된다



POST 요청이 발생하는 경우

  1. HTML form 태그를 사용하는 경우
  2. ajax에서 비동기화 검색
  3. ajax에서 post 방식으로 보내기



POST 요청 - Parameter 처리

HTTP 통신을 할 때 POST 요청은 요청 메시지의 body에 데이터를 넣어서 보낸다. 따라서 요청 메시지 body에 있는 데이터를 받아서 처리하는 부분이 필요하다.

PostController

@RestController
@RequestMapping("/api")
public class PostController {

  @PostMapping("/postMethod")
  public SearchParam postMethod(@RequestBody SearchParam searchParam) {
    return searchParam;
  }
}
  • @RequestBody SerchParam searchParam
    • RequestBody에 있는 값들을 searchParam 객체의 변수에 매핑해달라
  • GET 요청의 경우 Parameter를 객체로 받을 때 SearchParam searchParam만 명시해줬다



Request Form

Request Form에는 여러 형태가 있다. json, xml, multipart-form, text-plain 등.

이런 Form 중 하나를 produces를 사용해 설정할 수 있다

@PostMapping(value="/postMethod", produces = {"application-json"})



POST 요청 테스트 + json 데이터 처리하기

  • Google Chrome의 확장 프로그램 Restlet Client 툴을 통해서 request를 요청하고 response가 제대로 오는지 확인할 수 있다
  • POST 할 때 content-type은 보내는 데이터가 어떤 타입인지를 명시한다

흐름

POST 요청이 들어왔을 때 Spring은 헤더의 content-type을 확인하고, request body에 있는 json 값들을 searchParam 객체로 보내 각 변수에 매핑을 시켜준다. 매핑된 객체를 리턴할 때도 다시 객체에 접근해서 변수의 값들을 json 형태로 바꾼 다음에 리턴을 한다



HTTP 헤더 Content-Type

리소스의 타입을 나타내기 위해 MIME type, charset, boundary를 선택적으로 명시한다.

MIME type은 클라이언트에게 전송된 문서의 type을 나타내기 위해 사용한다. MIME type의 구조는 type/subtype으로 type은 카테고리를 나타내는데, 개별 타입과 멀티파트 타입 두 가지가 있다. 멀티파트 타입은 합성된 문서를 나타내는데 사용된다.

HTML <form>을 POST 메서드로 요청할 때, 요청의 Content-Type은 <form>의 enctype 속성에 의해 결정된다.

Content-Type이 멀티파트 타입일 경우 boundary를 꼭 명시해줘야 한다. boundary는 메시지의 멀티파트 경계선을 캡슐화하기 위해 사용한다.
