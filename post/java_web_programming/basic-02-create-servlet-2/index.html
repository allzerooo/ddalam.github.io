Java Web Programming

부스트코스 - 웹 프로그래밍에서 배운 내용을 정리


Servlet

  • Java 웹 애플리케이션의 구성 요소 중 동적인 처리를 하는 프로그램의 역할을 한다
  • 즉, 동적인 콘텐츠 제공을 위해서 프로그램을 수행, 페이지를 동적으로 만들어준다
  • WAS에서 동작하는 Java 클래스이다
  • HttpServlet 클래스를 상속받아야 한다


Servlet 작성 방법

웹을 개발할 때 서블릿을 직접 써서 개발하기 보다 프레임워크를 사용해서 개발하는 경우가 더 많다. 하지만 프레임워크들도 서블릿이 없이는 동작할 수 없다.

서블릿 작성 방법은 서블릿의 버전에 따라 두 가지로 나뉘어진다. 3.0 미만 버전에서는 어노테이션을 사용하고, 3.0 이상 버전에서는 web.xml 파일을 사용한다


Servlet version 3.0 이상

@WebServlet() 어노테이션을 사용해서 URL Mapping을 바꿀 수 있다

해당 URL로 요청되었을 때 서블릿이 실행된다


Servlet version 3.0 미만

다음과 같이 web.xml을 사용해서 URL Mapping을 바꿀 수 있다

<servlet>
  <description></description>
  <display-name>TenServlet</display-name>
  <servlet-name>TenServlet</servlet-name>
  <servlet-class>exam.TenServlet</servlet-class>
</servlet>
<servlet-mapping>
  <servlet-name>TenServlet</servlet-name>
  <url-pattern>/ten</url-pattern>
</servlet-mapping>


정리

어떤 URL 요청이 있을 때 그 요청이 동적인 처리를 필요로 한다면 Servlet을 실행해야 되고, 요청을 처리할 Servlet을 Mapping 해줘야 한다. 이러한 처리를 Servlet 3.0 이상에서는 어노테이션을 사용하고, Servlet 3.0 미만에서는 web.xml 파일을 사용한다. 실제 웹 개발을 할 때는 프레임워크를 사용하기 때문에 Servlet을 직접 써서 개발하지는 않지만 프레임워크들도 Servlet을 바탕으로 동적하기 때문에 이해하는 것이 좋다
