August 29, 2019
Java Web Programming

패스트캠퍼스 강의 내용을 정리


컨트롤러

사용자의 요청을 받기 위해 주소를 설정해야 한다.

컨트롤러는 주소들의 묶음을 모아놓은 것을 말한다


GET 요청 처리 방법

@RestController
@RequestMapping("/api")
public class GetController {

    @RequestMapping(method = RequestMethod.GET, path = "/getMethod")
    public String getRequest() {
        return "Hi getMethod";
    }
}
  • @RestController
    • spring에게 “해당 클래스를 RestController로 사용할거야”라고 알려주는 어노테이션
  • @RequestMapping("/api")
    • API 주소를 매핑하기 위해 사용하는 어노테이션
    • 어떤 주소를 받을 것인지 path를 지정
    • localhost:8080/api 까지 주소가 매핑된다
  • @RequestMapping(method = RequestMethod.GET, path = "/getMethod")
    • 사용자의 요청을 하나의 메서드로 받아들일 수 있다
    • @RequestMapping을 통해서 method는 어떤 타입으로 받을 것인지, 어떤 주소를 받을 것인지 지정
    • localhost:8080/api/getMethod 까지 주소가 매핑된다
  • return문은 브라우저에 출력된다


GET 요청 처리 방법 - Parameter가 있을 때

@RestController
@RequestMapping("/api")
public class GetController {

    @GetMapping("/getParameter")
    public String getParameter(@RequestParam String id, @RequestParam(name="password") String pwd) {
        System.out.println("id : " + id);
        System.out.println("password : " + pwd);

        return id+pwd;
    }

    @GetMapping("/getMultiParameter")
    public SearchParam getMultiParameter(SearchParam searchParam) {
        System.out.println(searchParam.getAccount());
        System.out.println(searchParam.getEmail());
        System.out.println(searchParam.getPage());

        return searchParam;
    }
}


처리해야 될 요청 1

localhost:8080/api/getParameter?id=1234&password=abcd
  • @GetMapping
    • GET 요청을 처리하는 어노테이션
    • @RequestMapping과 다르게 method를 지정하지 않아도 되고, 주소만 지정해주면 된다
  • @RequestParam(name="password") String pwd
    • @RequestParam은 Parameter를 받기 위해 사용하는 어노테이션
    • 만약 주소 창에서는 password로 값을 전해주는데 로직에서 다른 변수명을 사용해야 한다면, sping은 매칭하기 어렵기 때문에 name이라는 속성을 지정해준다. 즉, password로 들어오는 parameter를 pwd가 사용하겠다는 것.


처리해야 될 요청 2 - Parameter가 많아질 때

localhost:8080/api/getMultiParameter?account=abcd&email=abcd@gmail.com&page=10
  • Parameter를 객체로 받는다
  • SearchParam 클래스를 생성하고, getter/setter 메서드를 정의
  • return searchParam
    • 네트워크 통신을 할 때 JSON 형태로 주고 받는다
    • Spring Boot에서는 어떻게 처리하면 될까?
    • HTTP 통신에서는 JSON을 표준처럼 사용하기 때문에 Spring Boot는 기본적으로 jackson 라이브러리를 내장하고 있다
    • jackson 라이브러리는 객체를 JSON 형태로 바꿔준다
    • 객체를 return 한다는 것은 JSON 형태로 바꿔달라는 것과 같다
