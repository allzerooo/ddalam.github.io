2018년 KAKAO BLIND RECRUITMENT - 실패율

September 03, 2019

from operator import itemgetter

def solution(N, stages):
  answer = []

  stages.sort()
  l = len(stages)
  stages.append(501)
  now_stage = stages[0]
  not_clear_user = 0
  failure_rate = {}
  check = [False] * N
  i = 0
  while True:
    if stages[i] == now_stage:
      not_clear_user += 1
    else:
      reched_user = l - i + not_clear_user
      if now_stage != N+1:
        failure_rate[now_stage] = float(not_clear_user) / float(reched_user)
        check[now_stage-1] = True
      now_stage = stages[i]
      if now_stage == N+1:
        break
      not_clear_user = 1

      if stages[i] == 501:
        break

      i += 1


  temp = sorted(failure_rate.items(), key=itemgetter(0))
  temp = sorted(failure_rate.items(), key=itemgetter(1), reverse=True)

  for i in range(len(temp)):
    answer.append(temp[i][0])

  for i in range(N):
    if check[i] == False:
      answer.append(i+1)

  return answer


풀이

  1. stages를 정렬하고, 리스트의 마지막을 확인하기 위해 501을 추가한다.
  2. 현재 stage를 정해준다.
  3. 현재 stage와 같은 stage가 나올 동안 클리어 하지 못한 사용자를 세어준다.
  4. 현재 stage와 다른 stage가 나오면 현재 stage에 도달했던 모든 사용자를 구하고, 실패율을 계산한다. 그리고 check 리스트를 사용해서 확인한 stage를 기록한다.
  5. 현재 stage가 N+1이거나, stages에 확인하지 않은 값이 남아있는 동안 반복한다


수정할 점

  1. 리스트의 마지막을 확인하기 위해 501을 추가할 필요가 있나
  2. 실패율 계산을 현재 스테이지와 다른 스테이지를 만났을 때 처리하다 보니 마지막 나온 스테이지를 처리하기 어려워서 501과 여러 if문을 추가하게 되지 않았을까?


다른 사람 풀이

def solution(N, stages):
  result = {}
  denominator = len(stages)
  for stage in range(1, N+1):
    if denominator != 0:
      count = stages.count(stage)
      result[stage] = count / denominator
      denominator -= count
    else:
      result[stage] = 0
  return sorted(result, key=lambda x : result[x], reverse=True)
  • count() 함수를 사용
  • 각 stage별 실패율을 구할 때, stages에 있는 stage만 구한게 아니라 1~N까지 모든 stage별 실패율을 구하기 때문에 도달한 사용자가 0명인 경우에 대한 특별한 처리가 없이 가능하다
  • 결과 값을 정렬할 때 위와 같은 방법을 사용하면, 딕셔너리의 value 값을 기준으로 내림차순 정렬하고, key 값은 오름차순으로 정렬이 가능하다


비교

처음에 count() 함수를 쓸까 고민을 했었다. 그런데 count()의 시간 복잡도가 O(n)이면, stages의 최대 길이가 200,000이기 때문에 시간 초과가 나올 수 있다고 생각해서 다른 방법을 선택했다. 그래서 두 코드의 시간 차이가 궁금해졌다. count() 함수 외 다른 부분도 영향을 주었겠지만 ..

결과 : 총 27개의 테스트 케이스 중 4개 케이스는 같은 시간, 15개 케이스는 내가 작성한 코드가 빨랐고, 7개는 느리게 나왔다.

def solution(N, stages):

  usersPerStage = [0] * (N+1)
  for stage in stages:
    if stage == N+1:
      continue
    usersPerStage[stage] += 1

  failureRate = {}
  rechedUsers = len(stages)
  for stage in range(1, N+1):
    if rechedUsers != 0:
      failureRate[stage] = usersPerStage[stage] / rechedUsers
      rechedUsers -= usersPerStage[stage]
    else:
      failureRate[stage] = 0

  return sorted(failureRate, key=lambda x: failureRate[x], reverse=True)
