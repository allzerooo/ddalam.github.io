BOJ_2805 나무 자르기

September 11, 2019
Algorithm

문제


Code

n, m = map(int, input().split())
h = list(map(int, input().split()))

maxH = max(h)
leftbound = 0
rightbound = maxH
answer = 0
while leftbound <= rightbound:
    mid = (rightbound + leftbound) // 2
    takenLength = 0
    for i in range(n):
        if h[i] > mid:
            takenLength += (h[i] - mid)
    if takenLength >= m:
        leftbound = mid + 1
        answer = mid
    else:
        rightbound = mid - 1

print(answer)


풀이

가장 큰 나무의 높이를 maxH라고 하면, 절단기 높이로 가능한 구간을 0 ~ maxH로 설정할 수 있다. 나무의 수는 최대 1,000,000이고, 나무의 길이는 최대 2,000,000,000이기 때문에 절단기 높이가 0일 때부터 maxH일 때까지 모두 조사해보는 것은 불가능하다. 따라서 이진탐색을 사용한다.

  1. 두 구간의 가운데 값으로 절단기의 높이를 정한다
  2. 나무의 높이가 절단기의 높이보다 높으면 절단한다
  3. 절단한 나무 길이의 합이 찾고자 하는 길이보다 크거나 같으면 일단 정답으로 저장한다. 왜냐하면 문제에 적어도 M미터의 나무를 가져가겠다고 했고, 딱 M미터의 나무를 가지고 갈 수 없는 경우도 있기 때문이다. 그리고 구간을 가운데 값의 오른쪽 범위로 좁히기 위해 구간의 왼쪽 끝 위치를 가운데 값 + 1로 해준다. 나무 길이의 합이 찾고자 하는 길이보다 크거나 같다는 것은 절단기의 높이를 더 높여서 잘라야한다는 것이기 때문이다(절단기의 높이가 높아지면 가져갈 수 있는 나무의 길이는 작아진다)
  4. 절단한 나무 길이의 합이 찾고자 하는 길이보다 작으면 구간을 가운데 값의 왼쪽 범위로 좁히기 위해 구간의 오른쪽 끝 위치를 가운데 값 - 1로 해준다. 3번에서의 이유와 반대로 절단기의 높이를 낮춰야 가져갈 수 있는 나무의 길이는 커지기 때문이다
  5. 이러한 과정을 왼쪽 끝 위치가 오른쪽 끝 위치보다 작거나 같을 동안 반복한다. 왜냐하면 같아졌을 때도 절단기 높이는 1개의 값을 가질 수 있는 구간이기 때문이다
