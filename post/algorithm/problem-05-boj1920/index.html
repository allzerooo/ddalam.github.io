Code - 이진탐색

import sys
sys.setrecursionlimit(10000)

n = int(input())
a = list(map(int, input().split()))
m = int(input())
nums = list(map(int, input().split()))


def binary_search(target, start, end, data):
  if start > end:
    return 0

  mid = (start + end) // 2

  if data[mid] == target:
    return 1

  if data[mid] < target:
    start = mid + 1
  else:
    end = mid - 1

  return binary_search(target, start, end, data)


a.sort()

for i in range(m):
  print(binary_search(nums[i], 0, n-1, a))


Code - 집합

n = int(input())
a = list(map(int, input().split()))
m = int(input())
nums = list(map(int, input().split()))


set_a = set(a)
set_nums = set(nums)

intersection = set_a & set_nums

for i in range(m):
  if nums[i] in intersection:
    print(1)
  else:
    print(0)


풀이

이 문제는 입력 N과 M이 100,000으로 100,000 * 100,000 = 매우 크다 😇

1초 당 반복문 수행 횟수가 1억을 넘기면 시간 제한을 초과할 가능성이 있다. 따라서 탐색 시간을 줄여야 한다.

첫 번째 방법은 이진 탐색을 사용하는 것이다. 이진 탐색은 O(logN)의 시간 복잡도를 보장한다. 따라서 100,000 * log100,000으로 해결할 수 있다.

두 번째 방법은 집합을 사용하는 것이다. 두 리스트에서 교집합을 만들고, 각 수가 교집합 안에 포함됐는지 확인하는 방법이다. python의 교집합 연산은 시간 복잡도가 O(len(s)+len(t))이고, 집합에 element가 있는지 확인하는 연산은 O(1)이다.

백준 사이트에서 두 방법으로 채점을 해보니 이진 탐색을 사용한 방법은 692 ms, 집합을 사용한 방법은 228 ms 로 나왔다 🤔
