다형성이란

객체지향개념으로는 여러 가지 형태를 가질 수 있는 능력을 의미

  • 상속, 인터페이스를 이용해서 다형성을 구현


상속을 이용한 다형성

부모 클래스 타입의 참조변수로 자식 클래스의 인스턴스를 참조 → 하나의 부모 클래스 타입 참조변수로 여러개의 자식 클래스 인스턴스를 참조할 수 있는 유연한 표현이 가능


Exmaple 1. 부모 타입으로 배열을 선언

class Toy{ ... }
class Doll extends Toy{ ... }
class KeyHolder extends Toy{ ... }
class BubbleGum extends Toy{ ... }

class CatchToyMachine{
  private Toy[] toys;

  public CatchToyMachine() {
    toys = new Toy[10];
    for(int i = 0; i < toys.length; i++){
      Toy toy = null;
      if(i % 3 == 0){
        toy = new Doll();
      } else if(i % 3 == 1){
        toy = new KeyHolder();
      } else if(i % 3 == 2){
        toy = new BubbleGum();
      }
      toys[i] = toy;
    }
  }
}


Exmaple 2. 부모 타입 파라미터 사용

class ToyTest{
  public void testToy(Toy t){ ... }

  public static void main(String[] args){
    ToyTest test = new ToyTest();
    Toy t = new Toy();
    test.testToy(t);
  }
}

부모 타입으로 파라미터를 선언할 경우, 부모 클래스를 상속받는 자식 클래스의 객체 어떤 것이든 수용할 수 있다.


참조변수의 형변환

class Tv {
  boolean power;
  void power(){ ... }
}

class CationTv extends Tv {
  String text;
  void caption(){ ... }
}

Tv t = new CationTv();
t.caption()  // 컴파일에러 발생

컴파일러는 t의 변수 타입을 보고 t가 가리키는 객체가 Tv의 객체일 것으로 판단한다. 따라서 Tv 클래스에 없는 caption()을 호출하므로 컴파일에러가 발생한다. 이런 경우 참조변수의 형변환이 필요하다. (조상의 조상으로도 형변환이 가능하기 때문에 모든 참조변수는 모든 클래스의 조상인 Object 클래스 타입으로 형변환이 가능하다)

((CaptionTv)t).caption();


Up-casting : 조상클래스 타입의 참조변수로 자손클래스의 인스턴스를 참조(형변환 생략 가능)
Down-casting : 자손클래스 타입의 참조변수로 부모클래스의 인스턴스를 참조(형변환 생략 불가)

Tv t = (Tv)new CationTv() //(Tv)는 생략가능
// 위와 동일한 결과
CationTv c = new CaptionTv();
Tv t = (Tv)c;


참조변수의 형변환을 했는데 조상타입의 인스턴스를 자손타입의 참조변수로 참조하게 됐을 때

class Test{
  Tv t = new Tv();
  CaptionTv c = null;

  // 참조변수 t가 참조하고 있는 인스턴스를 참조변수 c가 참조하도록
  // 이 때 두 참조변수 타입이 다르므로 참조변수 t를 형변환
  c = (Tv)t;   // 실행 시 에러가 발생
}

위와 같은 상황을 예방하기 위해 참조변수가 가리키는 인스턴스 타입이 무엇인지 확인하는 것이 중요하다


instanceof

형변환을 할 때 참조변수가 가리키는 인스턴스 타입이 무엇인지 확인이 필요하며, 여러 참조변수가 있을 때 각각 가리키는 대상이 어떤 객체인지 모두 기억하기 힘들다. 참조변수가 가리키는 객체의 타입을 확인하기 위해 instanceof 연산자를 사용할 수 있다. 더 정확히 하면, instanceof 연산자는 참조변수가 검사한 타입으로 형변환이 가능한지를 판단하는 연산자이다. 따라서 아래와 같은 문제가 발생할 수 있다.

Tv[] arr = new Tv[3];
arr[0] = new Tv();
arr[1] = new CaptionTv();
arr[2] = new CaptionTv();

for(int i = 0; i < arr.length; i++){
  if(arr[i] instanceof Tv){
    System.out.println(i + "번째는 TV");
  }else if(arr[i] instanceof CaptionTv){
    System.out.println(i + "번째는 Caption TV");
  }
}
0번째는 TV
1번째는 TV
2번째는 TV

이에 대한 대안으로 참조변수.getClass().getName()을 사용할 수 있다.
